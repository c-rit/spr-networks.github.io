SPR: A Secure Programmable Router

The aim of SPR is to make home networks much more secure, programmable, and reliable.

Home router software today limits innovation because it is not easy for developers to deploy custom code, modify core networking, or maintain their software. On top of that, the home networking model has not properly adapted to the security and privacy needs of today's internet members. It's no longer just about keeping the internet out, it's also about containing the devices on the network within, because users have little if any control over their devices' software.

Most home routers are absolutely riddled with security holes that allow attackers to break into the router once they are on the network, and sometimes even remotely from across the internet.

Even without the software security flaws, the networks are very poorly designed and provide very little defense in depth, if any. Attackers can readily jump between the guest wifi network and the main network, impersonate other devices with a compromised WiFi passphrase, and carry out spoofing attacks to intercept traffic. In today's standard home network the security is, at best, only as strong as the weakest client device on the network.

SPR simply enables users to do better than today's status quo. It lets users run a hardened, secure network without restrictive drawbacks. It lets users connect their consumer electronics to the internet with the peace of mind that doing so does not weaken their home network security.

The Network Design​

  • Default deny. Devices should be explicitly marked for communicating to the internet, accessing DNS, or intranet communications either by a general LAN group or custom groups.

  • Each device on the network is sequestered into a tiny subnet, and all devices must communicate through the router to communicate with one another. Groups are used to inform nftable verdict maps about a device's network access, by MAC address, virtual interface, and IP address. By trading off negligible latency and additional packet processing for better control, the router arbitrates communication between intranet devices. The verdict maps, similar to ipsets, provide scalable groups without exponential firewall rules.

  • Multicast support with a proxy. Great things can happen with multicast, and zeroconf and mDNS are both prevalent and a requisite for home networking. However, these require being on the same subnet to work or a router that supports IGMP forwarding or a proxy. SPR has a userland proxy to transparently relay multicast traffic between devices and across network interfaces.

  • Per-device WiFi passphrases create a source of identity for devices. This enables a high degree of confidence about where packets come from and where they can go. This happens without installing agents, custom software, or certificates on devices making a router runing SPR compatible with all WiFi devices.

The Software Design​

  • Fully Open Source

  • Networking services are split across docker services. Containers are used for separation of concerns and in the future could contribute to system sandboxing to mitigate security risks.

  • Where possible, memory-safe languages with good security track records are preferred over native code such as C or code that is difficult to write securely, like PHP.

The Service Listing​

The following services are currently included with SPR:

  • api ⇨ API for frontends and CLIs to manage connected devices, network groups, and firewall rules
  • base ⇨ Networking initialization and configuration at startup
  • dhcp ⇨ DHCP Server
  • dhcp_client ⇨ DHCP Client for requesting IP addresses from uplink
  • dns ⇨ DNS Service, running CoreDNS
  • wifid ⇨ Wifi Base Station Service, running hostapd
  • multicast_udp_proxy ⇨ A multicast proxy to support multicast networking for usability
  • ppp ⇨ A Point-to-Point protocol service for authenticating and requesting IP addresses from uplink
  • watchdog ⇨ A failure mode detection service for restarting the router in case of crashes
  • wireguard ⇨ Configuration service for running a Wireguard VPN
