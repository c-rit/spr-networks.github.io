Raspberry Pi 4 Model B Setup Guide

Configuration​

The current setup assumes you'll be using a raspberry pi model 4b with an mt76 based wireless dongle (wlan1). Optionally an additional usb ethernet dongle (eth1) can be connected to a switch for additional wired devices. The built-in ethernet port of the raspberry pi (eth0) is connected to upstream/WAN/internet

Hardware requirements:

  • At least a 16GB SD card is required, and a USB flash drive works best for development.
  • A WiFi Dongle for better performance and WPA3 support. Both (https://www.alfa.com.tw/products/awus036acm and https://www.netgear.com/home/wifi/adapters/a6210/ have been tested to be good and both use the mt76x2u driver)
  • (Optional) A USB WiFi Dongle for additional LAN devices since the built-in ethernet card (eth0) will be used for the WAN. https://www.tp-link.com/us/home-networking/usb-converter/ue300/ has been tested to be near line speed.

Base System Setup​

  1. Set up the pi with ubuntu server 64-bit https://ubuntu.com/download/raspberry-pi/thank-you?version=21.10&architecture=server-arm64+raspi
# Example from mac
$ xzcat ubuntu-21.10-preinstalled-server-arm64+raspi.img.xz | dd of=/dev/rdisk2 bs=$[1024*1024]

1.a. Tune the ubuntu config (see base/setup.sh)

# On the booted pi (see and run base/setup.sh)
git clone https://github.com/spr-networks/super
cd super
#if using a flash drive
sudo base/setup.sh
#or if using an sdcard
# base/setup-sdcard.sh #this setup will reduce writes from log files
reboot

Configuring the project​

  1. Inside the super directory, copy base/template_configs to configs/
# cd super
# cp -R base/template_configs configs

  1. Modify configs/base/config.sh to set an SSID_NAME or change other options. Ensure that WANIF matches the name of the outbound interface if not using the builtin ethernet port.

  2. Set an api username and password for the web UI. The UI will be reachable on port 80 from connected devices.

mkdir -p state/api/
echo {\"admin\" : \"your password goes here\"} > state/api/auth_users
  1. (Optional)

For testing, it is likely that the API/SSH should also be reached from the WAN interface. Modify configs/base/nft_rules.sh to remove port 80 and 22. The base container must be restarted (if already running) for the change to take effect.

The firewall rule to change looks like: iifname $WANIF tcp dport {22, 5201, 80} counter jump DROPLOGINP

To remove 22/80 from being dropped: iifname $WANIF tcp dport {5201} counter jump DROPLOGINP

  1. (Optional)

It is possible to manually configure zones and stations ahead of time, without using the web interface. For WPA3, use "sae" type and for WPA2 use "wpa2". See configs/wifi/psks.json

  1. (Optional)

Lastly, manually add those devices to zones in configs/zones/zones.json The default zones are:

# No access to anything
- isolated
# Allows DNS query access
- dns
# Allows internet/WAN forwarding
- wan
# Allows full LAN access
- lan

Running the project using prebuilt containers​

Testing containers are built and published to Github's Container Registry, and can be run without building on the device. If you would instead like to build everything from source, proceed to the next section.

./configs/scripts/gen_coredhcp_yaml.sh > configs/dhcp/coredhcp.yml
./configs/scripts/gen_hostapd.sh > configs/wifi/hostapd.conf
./configs/scripts/gen_watchdog.sh  > configs/watchdog/watchdog.conf

./run_prebuilt.sh

To see logs:

docker-compose logs -f

Building the project​

All source code can be built on the pi. Because the system uses nftables exclusively, and docker is made to work with iptables, you will need to set up minimal forwarding rules for docker to work on the first install.

#fix nftables
root@ubuntu:/home/ubuntu/super# ./base/docker_nftables_setup.sh

And build.

root@ubuntu:/home/ubuntu/super# ./build_docker_compose.sh

If something went wrong with with a cached stage, it is possible to specify --no-cache. A specific service can also be passed as an argument (./build_docker_compose.sh api)

To run the local build:

./run_docker_compose.sh

Additional Notes​

Check dns-Corefile to tweak DNS server configuration as well as the hostapd settings in configs/gen_hostapd.sh

It is possible to use SPR's dns server for the host system's DNS requests too, by updating the resolv.conf file.

root@ubuntu:/home/ubuntu/super# echo -e "nameserver 127.0.0.1\nnameserver 1.1.1.1" > /etc/resolv.conf

Using a different wireless dongle​

For using the built-in wireless or a different dongle, the hostapd configuration may need to be modified in configs/gen_hostapd.sh. Note that if the built-in wireless is to be used, WPA3 is not currently available without additional broadcom firmware patches. The next raspberry pi os releases should include these by default.
